Complex Buttons

Example Description Usage
Stateful

Markup:

<button rel="stateful" data-loading-text="loading..." class="btn">Button</button>

Script:

$(…).button('loading');

Sets button state to loading—disables button and swaps text to loading text. Loading text should be defined on the button element using the data attribute data-loading-text.

NOTE: Firefox persists the disabled state across page loads. A workaround for this is to use autocomplete="off".

$(…).button('reset');

Resets button state—swaps text to original text.

Script Requirements:
  • bootstrap/bootstrap-button
Toggle

Markup:

<button class="btn btn-primary" data-toggle="button">Single Toggle</button>

Script:

$(…).button('toggle');

Toggles push state. Gives the button the appearance that it has been activated.

The data-toggle attribute enables auto-toggling.

Script Requirements:
  • bootstrap/bootstrap-button

 Button Dropdowns

Use any button to trigger a dropdown menu by placing it within a .btn-group and providing the proper menu markup.

To quickly add dropdown functionality to any element just add data-toggle="dropdown" and any valid bootstrap dropdown will automatically be activated.

<div class="btn-group">
  <button class="btn dropdown-toggle" data-toggle="dropdown">Action <i class="icon-ico_chevron_down_lg></i></button>
  <ul class="dropdown-menu">
    <li><a href="#">Action</a></li>
    <li><a href="#">Another action</a></li>
    <li><a href="#">Something else here</a></li>
    <li class="divider"></li>
    <li><a href="#">Separated link</a></li>
  </ul>
</div>

You may optionally target a specific dropdown by using data-target="#id" or href="#fat".

NOTE: In some cases—like mobile—dropdown menus will extend outside the viewport. You need to resolve the alignment manually or with custom javascript.

  • bootstrap/bootstrap-dropdown

Navigation

Primary Navigation

Masthead & Navigation

The header contains primary navigation links to all top-level pages and masthead elements including:

  • Application ID composed of the GE monogram and the site title
  • Notification controls, including Alerts, Messages, and potentially Tasks
  • Account details, including links to Settings and Help
  • The site's main search field

Usage Notes:

  • Notification controls are optional
  • Account details are optional
  • Less than 7 top-level sections are recommended, but if number of links exceeds available screen width, a More link can be inserted that contains overflow items

Flexible Dropdown

Similar to a mega-dropdown, the flexible dropdown presents multiple secondary and tertiary navigation elements as the child of a primary navigation element, but the display area of the flexible dropdown is sized to fit available content.

Usage Notes:

  • Dropdown displayed on click
  • Dropdown size changes to accommodate the amount of content available
  • On a mobile device, tap on a top-level navigation item to expose the dropdown menu. Tapping on a menu item will display the related item or page.

Script:

$(…).tooltip(options);

Options:

Name Type Default Description
animation boolean true apply a css fade transition to the tooltip
placement string|function 'top' how to position the tooltip - top | bottom | left | right
selector string false If a selector is provided, tooltip objects will be delegated to the specified targets.
title string | function '' default title value if `title` tag isn't present
trigger string 'hover' how tooltip is triggered - hover | focus | manual
delay number | object 0

delay showing and hiding the tooltip (ms) - does not apply to manual trigger type

If a number is supplied, delay is applied to both hide/show

Object structure is: delay: { show: 500, hide: 100 }

All of the above options may be applied independently as data- attributes.

NOTE: For performance reasons, tooltips are opt-in. They must be manually initialized to function.

Script Requirements:

Highcharts Charts

Series Charts

A series chart contains a single value for each point along the primary axis. Multiple series charts sharing an axis can be shown together to allow visual comparisons.

The following series chart examples share this core configuration: average monthly temperature (a series) for two locations.

var seriesConfig = {
  title: { text: 'Monthly Average Temperature' },
  subtitle: { text: 'Source: <a href="http://WorldClimate.com">WorldClimate.com</a>' },
  series: [{
    name: 'Tokyo',
    data: [7.0, 6.9, 9.5, 14.5, 18.4, 21.5, 25.2, 26.5, 23.3, 18.3, 13.9, 9.6]
  }, {
    name: 'London',
    data: [3.9, 4.2, 5.7, 8.5, 11.9, 15.2, 17.0, 16.6, 14.2, 10.3, 6.6, 4.8]
  }],
  xAxis: {
    title: {
      text: 'Month'
    },
    categories: ['Jan', 'Feb', 'Mar', 'Apr', 'May', 'Jun', 'Jul', 'Aug', 'Sep', 'Oct', 'Nov', 'Dec']
  },
  yAxis: {
    title: {
      text: 'Temperature (°C)'
    }
  }
};

See highcharts.com for information about highcharts configuration.

Radial Charts

Radial charts divide a circle according to the value of each data point in a series. Values will be normalized (they do not have to add up to 100%.)

The following charts share this core configuration:

d3 Charts

Standard chart types are defined with simple declarative markup, and rendered as SVG graphics with d3.

To invoke visualizations, first use a <div></div> element with the data-visualization attribute set to the appropriate visualization type. Additional data- attributes are used by each type to declare the values for rendering.

<div data-visualization="bar"></div>

To initialize the visualization, first require the declarative visualizations module to import the necessary code libraries.

require(['declarative-visualizations'], function(visualizations) {
            ...
            });

In the require callback, invoke the visualization method on each of the matched elements and apply any additional functionality as needed.

$('div[data-visualization="gauge"]').each(function(i, element) {
            var gauge = visualizations.gauge(element);
            });

By changing the data- attributes on the tag and then calling update you can redraw the visualization.

var gauge = visualizations.gauge(element);

            // ... change data-* attributes ...

            gauge.update();

Description

A bar chart graphic.

HTML5 data- attributes are used to control parameters of the graphic.

Name Description
data-duration Animation speed in milliseconds of visualization for initial render and subsequent updates. (optional)
data-visualization Type of graphic. Should be "bar".
data-max Maximum value.
data-min Minimum value. Defaults to 0. (optional)
data-target Target URL for optional JSON source. Will ignore other `data-` attributes if specified. (optional)
data-threshold Threshold value. Defines the position of the white marker on the visualization. (optional)
data-value Current value. Defines the size of the colored area.
